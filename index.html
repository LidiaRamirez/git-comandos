GIT

Para utilizar git en windows se debe descargar desde su página oficial: https://git-scm.com/. En el caso de otros sistemas operativos se puede guiar de este link.

Pero antes de comenzar a utilizar GIT, se recomienda saber comandos para el uso de la terminal

Comandos en terminal en windows

• Navegar por carpetas

cd nombre-archivo

• Salir de carpeta

cd ..

• Crear carpetas

mkdir nombre-carpeta

• Visualizar carpetas y archivos

ls

• Limpiar consola, que en realidad al hacer scroll el contenido sigue allí

clear

• Crear archivos

touch nombre-archivo.extension

• Borrar archivos

rm nombre-archivo.extension

• Borrar carpeta

rm -rf nombre-carpeta

• Editar archivo

vim nombre-archivo.extension

• Escribir, Guardar (w) y cerrar archivo (q) editado, dar esc

:wq

• Para salir de editar archivo

:q!

• Generar llave pública

Si es primera vez, se debe crear carpeta .ssh en carpeta de usuario del sistema operativo

mkdir .ssh

Luego ubicarse en carpeta .ssh

cd .ssh

Y el comando para generar llave pública:

ssh-keygen -t rsa ó ssh-keygen.exe

Comandos relevantes de git

• Versión de Git

git version ó git --version

• Configuración global

git config --global user.email correo@mail.com
git config --global user.name "nombre usuario"

• Mostrar información de configuración

git config --list

• Crear repositorio Git (Paso 1)

git init

• Estado del repositorio

git status

• Agregar archivos al directorio local (Paso 2)

Agregar un archivo específico

git add nombre-archivo.extension

Agregar todos los archivos

git add . ó git add --all

• Eliminar del directorio local

git rm --cached nombre-archivo.extension

Forzar un remove definitivamente

git rm -f index.css

• Confirmar cambios (Paso 3)

git commit -m "mensaje descriptivo"

Ejm: git commit -m "Iniciando proyecto"

• Agregar un archivo al último commit

git add archivo-olvidado.extension git commit --amend

• Agregar un archivo y reemplazar último commit

git add archivo-olvidado.extension git commit --amend 'Nuevo mensaje'

Si aparece el editor, solo colocar :wq! para salir

• Etiquetas (tag) anotadas para Git

git tag -a version-numero -m 'mensaje'

Ejm: git tag -a v1.5 -m 'stable version of the project'

• Visualizar todos los tag

git tag -l

Conectar con repositorio remoto

git remote add origin url-repositorio

• Visualizar la URL asociada a repositorio remoto

git remote -v

• Subir cambios a un repositorio remoto (Paso 4)

git push origin nombre-rama
git push origin master

• Actualizar las referencias remotas

git pull origin nombre-rama

• Historial de commits

git log

• Crear una rama

git checkout -b nombre-rama

• Cambiar de una rama a otra

git checkout nombre-rama

• Deshacer cambios de un archivo

git checkout nombre-archivo.extension

• Borrar una rama

git branch -D nombre-rama

• Cambiar de rama, pero sin confirmar cambios realizados hasta entonces

git stash

• Aplicar cambios de un guardado y lo retira inmediatamente de la pila.

git stash pop

• Clonar un repositorio

git clone url-repositorio

• Traer todas las ramas remotas a local

git fetch origin

• Visualizar todas las ramas

git branch git branch -a

• Fusionar ramas

Para ello hay que ubicarse en la rama que queremos dar merge para traer cambios (si hubieran) de otra rama

git merge otra-rama

• Comandos para publicar página en Github

git checkout -b gh-pages
git pull origin master
git push origin gh-pages
