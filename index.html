GIT

Para utilizar git se debe descargar desde su pagina oficial: https://git-scm.com/

Pero antes de comenzar a utilizar GIT, se recomienda saber comandos para el uso de la terminal

Comandos en terminal en windows

• Navegar por carpetas

cd nombre-archivo

• Salir de carpeta

cd ..

• Crear carpetas

mkdir nombre-carpeta

• Visualizar carpetas y archivos

ls

• Limpiar consola, que en realidad al hacer scroll el contenido sigue allí

clear

• Crear archivos

touch nombre-archivo.extension

• Borrar archivos

rm nombre-archivo.extension

• Borrar carpeta

rm -rf nombre-carpeta

• Editar archivo

vim nombre-archivo.extension

• Escribir, Guardar (w) y cerrar archivo (q) editado, dar esc

:wq

• Para salir de editar archivo

:q!

Comandos relevantes de git

Versión de Git

git version ó git --version

Configuración global

git config --global user.email correo@hotmail.com
git config --global user.name nombre
git config --global user.name "nombre usuario"

Crear repositorio Git (Paso 1)

git init

Inicialmente esta en working directory

Estado del repositorio

git status

Agregar archivos a stage (Paso 2)

Agregar un archivo específico

git add nombre-archivo.extension

Agregar todos los archivos

git add .

Eliminar de stage

git rm --cached nombre-archivo.extension

Forzar un remove definitivamente

git rm -f index.css

Confirmar cambios (Paso 3)

git commit -m "mensaje descriptivo"

Si queremos agregar un archivo a un último commit

git add archivo-olvidado.extension git commit --amend

Si queremos agregar un archivo y reemplazar último mensaje

git add archivo-olvidado.extension git commit --amend 'Nuevo mensaje'

* Aparece el editor, solo colocar :wq! para salir

Subir cambios a un repositorio remoto (Paso 4)

git push origin nombre-rama
git push origin master

Actualizar las referencias remotas

git pull origin nombre-rama

Etiquetas para GIT

Para el ultimo commit, con anotación

git tag -a version0.5 -m 'version estable del proyecto'

Para visualizar todos los tag

git tag -l

Traer todas las ramas remotas a local

git fetch origin

Visualizar todas las ramas

git branch git branch -a

Pasos para publicar página en github

git checkout -b gh-pages
git pull origin master
git push origin gh-pages

* Una opción de consola personalizada es hyper, se puede descargar desde su pagina oficial: https://hyper.is/
